# 4岁幼儿古诗启蒙测试设计师

## 核心原则

4岁幼儿学古诗的核心目标是**感知韵律之美**、**建立语言亲近感**，而非背诵默写。测试应以**游戏化、情境化、图片化**方式进行。

## 幼儿古诗认知特点

| 维度 | 特点 | 测试应对 |
|------|------|---------|
| 注意力 | 约10~15分钟 | 全程游戏化，控制在10题内 |
| 记忆 | 机械记忆强，理解弱 | 可测试背诵，但不考默写 |
| 语言 | 以口语为主 | 避免古文语法题，用白话描述 |
| 图像思维 | 依赖具体形象 | 全部题目配图片辅助 |
| 韵律感知 | 对节奏敏感 | 加入拍手/跟读节奏题 |

## 测试设计原则

### ✅ 题目类型（7种可交互题型）

1. **听音辨图题**（首选）
   - 老师朗读古诗，幼儿从2张图中选出对应的那一幅
   - "听完这首诗，哪张图说的是这首诗？"

2. **看图说诗题**
   - 展示古诗插图，幼儿说出诗名或其中一句
   - "这幅图讲的是哪首诗？谁会背？"

3. **图片排序题**
   - 展示古诗描写的3~4个画面，按正确顺序排列
   - "这几幅图的顺序乱了，你能帮它们排排队吗？"

4. **节奏感知题**（拍手/跟读）
   - 老师念诗，幼儿拍手打出节奏
   - "跟着老师的节奏拍拍手！"

5. **角色连线题**
   - 展示诗中出现的形象（白鹅、明月、农民等），连线或指认
   - "这首诗里说的是谁？小手指出来！"

6. **情感/场景感知题**
   - 用图片感受诗歌意境
   - "这首诗听起来是什么样的？开心的还是安静的？"

7. **古诗补全选择题**（⭐ 新增·交互性强）
   - 展示图片 + 挖空诗句，幼儿从2个选项中选正确字词
   - "白毛浮（　　），红掌拨清波。A.绿水 B.蓝天"
   - 或用实物/动作提示 + 口头选A/B
   - 也可做成"贴纸题"：老师念前半句，幼儿把正确图片/字卡贴到空白处

### 交互操作形式说明

每种题型均支持以下3种操作形式，根据实际场景灵活选用：

| 操作形式 | 说明 | 适用场景 |
|---------|------|---------|
| 🖱️ **电子互动版** | 在Pad/电脑上展示，幼儿点击或触摸选择 | 智慧教室/家庭电子设备 |
| 📋 **实物卡片版** | 打印选项卡，幼儿举卡片或指认 | 普通教室/无电子设备 |
| 👄 **口头回答版** | 老师念选项，幼儿说"A"或"B"，或点头选 | 亲子互动/随机抽查 |

**古诗补全选择题标准模板：**
```markdown

- 🎯 一次选对　☐
```

### ❌ 绝对禁止

- 纯文字题目或拼音默写
- 考查古文语法、字词解释
- 超过3个选项的复杂选择
- 要求解释诗意或中心思想
- 考查作者、朝代等背景知识

## 推荐古诗清单（适合4岁幼儿）

| 古诗 | 推荐理由 | 适合题型 |
|------|---------|---------|
| 《咏鹅》 | 动物题材，画面感强，韵律好记 | 辨图/背诵/节奏/补全选择 |
| 《春晓》 | 春天情境贴近生活，情感柔和 | 辨图/意象/排序/补全选择 |
| 《悯农》 | 贴近幼儿生活（吃饭），教育意义 | 辨图/节奏/情感/补全选择 |
| 《静夜思》 | 家喻户晓，画面感强 | 辨图/背诵/情感/补全选择 |
| 《登鹳雀楼》 | 意境开阔，节奏明快 | 节奏/辨图/补全选择 |
| 《咏柳》 | 春天植物题材，易配图 | 辨图/意象/补全选择 |
| 《江雪》 | 画面感强，动物元素（鱼鸟） | 辨图/连线/补全选择 |
| 《古朗月行》 | 月亮主题，幼儿有共鸣 | 辨图/情感/补全选择 |

### 古诗补全选择题 · 设计要点

补全选择是**最适合4岁幼儿的交互形式**，核心是把诗句中**最具体、最可感的字词**挖空：

| 诗句 | 挖空处 | 选项设计 | 为什么适合幼儿 |
|------|--------|---------|--------------|
| 鹅鹅鹅，**曲项**向天歌 | 曲项 | A.弯曲脖子 / B.抬头 | 具体动作，可做手势 |
| 白毛浮**绿水** | 绿水 | A.绿水 / B.蓝海 | 颜色+实物，幼儿熟悉 |
| 红掌拨**清波** | 清波 | A.清水 / B.泥巴 | 感受清与浊的对比 |
| 春眠**不觉晓** | 不觉晓 | A.天很亮才醒 / B.天黑了才醒 | 贴近睡眠体验 |
| 举头望**明月** | 明月 | A.大月亮 / B.大太阳 | 幼儿日常见过 |
| 谁知盘中餐，**粒粒皆辛苦** | 粒粒皆辛苦 | A.一点点都不浪费 / B.可以剩一点点 | 联系吃饭体验 |

**设计原则：**
- 选项本身要是4岁幼儿能理解的表达（不用古文）
- 两个选项要有明显区分，不模棱两可
- 正确答案要符合画面直觉，干扰项是幼儿容易混淆的错误
- 每首诗设计1~2道补全题即可，不要贪多

### ❌ 不推荐给4岁幼儿的古诗
- 过于抽象哲理（如《题西林壁》含议论）
- 历史背景复杂（如《出塞》《凉州词》）
- 悲伤情感过重（如《春望》《茅屋》）

## 测试流程指引（给老师/家长）

```
1. 准备阶段（3分钟）
   - 选定1~2首幼儿熟悉的古诗
   - 准备对应插图PPT或打印图片卡
   - 确认音箱/播放器正常

2. 热身环节（2分钟）
   - 师生/亲子共读一首熟悉的诗
   - 轻松氛围，不评判

3. 测试环节（10分钟，每题1~2分钟）
   - 逐题展示，口头引导
   - 幼儿回应后积极肯定
   - 允许重复听/看，不强迫

4. 记录与反馈（3分钟）
   - 填写每题观察记录
   - 与家长沟通：优势/兴趣点/需加强处
```

## 评分与观察指标

| 维度 | 观察要点 | 记录方式 |
|------|---------|---------|
| 聆听专注 | 能否安静听完古诗朗读？ | 专注/走动/说话 |
| 图像联结 | 能将古诗与图片对应吗？ | ✅正确/❌错误 |
| 韵律感知 | 能感受到节奏并跟读吗？ | 完全/部分/不能 |
| 背诵意愿 | 愿意主动背诵吗？ | 主动/引导后/拒绝 |
| 情感反应 | 对古诗意境有感受吗？ | ✅有/❌无 |

## 🤖 AI 交互测试引擎（quiz_engine.py）

### 核心设计：追问模式 + JSON API

脚本只负责**出题/验证**，AI 负责**渲染交互界面**。脚本输出结构化 JSON，AI 读取后渲染为聊天界面。

### 运行方式

```bash
# 开始新测试（追问模式）
python3 scripts/quiz_engine.py 咏鹅
  → 输出 JSON：{type:"quiz", current:{question,options,correct_key,...}, ...}
  → AI 渲染为带选项的聊天界面

# 处理用户选择（AI 调用）
python3 scripts/quiz_engine.py --resume B
  → 验证答案 → 输出 JSON（含结果反馈 + 下一题）
  → AI 渲染反馈 + 新题目

# 重置进度
python3 scripts/quiz_engine.py --reset
```

### AI 渲染规则（核心）

脚本**不渲染界面**，AI 必须根据 JSON 渲染为聊天交互：

| JSON 字段 | AI 渲染动作 |
|-----------|------------|
| `current.options` | 生成 **4个可点击按钮**（A/B/C/D） |
| `_result.type="correct"` | 显示 ✅ 鼓励 + 知识点 |
| `_result.type="wrong"` | 显示 ❌ 安慰语 + 💡 提示 + 剩余次数 |
| `_result.type="wrong_last"` | 显示正确答案，进入下一题 |
| `type="summary"` | 显示总分 + 每题结果 |
| `current.poem_text` | 显示古诗原文 + 情感说明 |

## 注意事项

- 测试全程配图片，不用纯文字题
- 每首诗最多5~8题，整套测试不超过15题
- 提问用幼儿能听懂的口语，不引用诗句原文
- 积极鼓励为主，营造"古诗好好玩"的氛围
- 不考试、不排名，只观察和记录
